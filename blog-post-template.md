## 2-1️⃣ 목차 (Table of Contents) 구성 가이드

### 📋 목차 위치 및 스타일
- **위치**: Hook 직후, 본문 시작 전
- **스타일**: 박스 형태로 강조 표시

### 📝 목차 작성 예시 (H2만 표시)

```markdown
## 📋 목차
- [1. 하드코트 — 속도와 공격의 코트](#1-하드코트--속도와-공격의-코트)
- [2. 클레이코트 — 인내와 전술의 코트](#2-클레이코트--인내와-전술의-코트)
- [3. 당신의 스타일로 선택하기](#3-당신의-스타일로-선택하기)
- [4. 자주 묻는 질문 (AEO)](#4-자주-묻는-질문-aeo)
- [5. 코치의 이야기](#5-코치의-이야기)
- [6. 함께 보면 좋은 콘텐츠](#6-함께-보면-좋은-콘텐츠)
```

**중요**: 목차에는 **H2 섹션만** 포함하고, H3, H4 등 하위 섹션은 제외합니다.

### 🎯 목차 최적화 원칙 (H2 전용)

| 항목 | 기준 | 설명 |
|------|------|------|
| **H2 개수** | 4~6개 | 너무 많으면 산만, 적으면 단조로움 |
| **제목 길이** | 8~15자 | 간결하면서도 의미 전달 |
| **키워드 포함** | 각 H2에 1개씩 | SEO + 사용자 이해도 향상 |
| **순서** | 중요도 순 | 핵심 내용부터 배치 |
| **H2만 포함** | 하위 섹션 제외 | H3, H4 등은 목차에서 제외 |

### 💡 목차 작성 팁
1. **H2 제목을 먼저 작성** → 목차 자동 생성
2. **앵커 링크는 한글 제목 그대로** 사용 (URL 인코딩 자동 처리)
3. **모바일에서도 접기/펼치기** 가능하도록 구현
4. **현재 읽고 있는 섹션 하이라이트** 표시

## 4️⃣ AEO (Answer Engine Optimization) 설계

> ✅ AEO는 **FAQ 구조 + Q형 문장 2회 + 직답형 문장 2회**로 구성됩니다.

예시:

```
**Q. 스트링 텐션은 몇 파운드가 적당할까요?**  
A. 초보는 48~50lbs, 중급은 52~55lbs가 평균입니다. 다만 스윙 속도에 따라 달라집니다.
```

이 구조는 Google SGE, Bing Copilot, Naver AiR 등에서 **“직답 추출형 문단”**으로 인식됩니다.

## 8️⃣ 시각 구성 및 접근성 (최종 업데이트)

| 항목       | 기준                      |
| -------- | ----------------------- |
| 본문 최대 너비 | 720px                   |
| 폰트       | Inter / Pretendard      |
| 색상 체계   | **심플 라이트 테마** (배경: 흰색/연한회색, 텍스트: 진한회색) |
| 다크모드     | 자동 전환, 대비 ≥ 4.5:1       |
| 목차       | 모바일 접힘 (최대 40vh)        |
| 버튼       | 반응형, aria-label 포함      |
| 텍스트 강조  | `<strong className="text-gray-900">` 필수 적용 |

### 🎨 색상 및 가독성 규칙 (고대비 버전)

#### 🎯 **핵심 원칙: 최대한 명확하게**
- **배경색**: 흰색(`bg-white`) 또는 연한 회색(`bg-gray-50`)만 사용
- **텍스트 색상**: 진한 검정 계열로 통일하여 가독성 극대화
- **강조 색상**: 최소한의 색상으로 중요 부분만 구분

#### ❌ **완전 금지사항**
- **다양한 배경색 사용 금지**: `bg-orange-50`, `bg-blue-50`, `bg-green-50` 등 사용 금지
- **연한 텍스트 색상 금지**: `text-gray-400`, `text-gray-500`, `text-gray-600` 사용 금지
- **투명도 적용 금지**: `/50`, `/30` 등 투명도 사용 금지
- **색상별 구분 금지**: 섹션별로 다른 색상 사용 금지

#### ✅ **권장 색상 체계 (고대비)**
```css
/* 기본 텍스트 - 고대비 */
text-gray-900  /* 제목, 중요 텍스트 (최고 대비) */
text-gray-800  /* 부제목, 강조 텍스트 */
text-gray-700  /* 본문 텍스트 (고대비) */

/* 배경 - 단순화 */
bg-white       /* 기본 배경 */
bg-gray-50     /* 섹션 구분용 (최소한만 사용) */

/* 테두리 - 명확한 구분 */
border-gray-300 /* 기본 테두리 */
border-gray-400 /* 강조 테두리 */

/* 강조 색상 - 최소한만 사용 */
text-blue-600  /* 링크 텍스트 */
text-green-600 /* 성공/완료 텍스트 */
text-red-600   /* 경고/주의 텍스트 */
```

#### 🔧 **텍스트 강조 규칙 (고대비)**
- **`<strong>` 태그**: 반드시 `text-gray-900` 클래스 추가
- **코치 팁**: `<strong className="text-gray-900">코치 팁:</strong>`
- **중요 포인트**: `<span className="text-gray-800 font-semibold">중요:</span>`
- **링크 텍스트**: `<a className="text-blue-600 hover:text-blue-800">링크</a>`
- **경고/주의**: `<span className="text-red-600 font-semibold">주의:</span>`
- **성공/완료**: `<span className="text-green-600 font-semibold">완료:</span>`

### 📝 관리자 주석 처리 규칙

#### 🚫 독자에게 노출되면 안 되는 주석
- **GEO**: 지역 최적화 관련 관리자 주석
- **AEO**: 답변 엔진 최적화 관련 관리자 주석  
- **SEO**: 검색 엔진 최적화 관련 관리자 주석

#### ✅ 처리 방법
1. **코드에서 제거**: 최종 렌더링 시 주석 제거
2. **숨김 처리**: CSS `display: none` 또는 `visibility: hidden`
3. **개발자 모드에서만 표시**: `process.env.NODE_ENV === 'development'` 조건부 렌더링

#### 📋 주석 제거 체크리스트
- [ ] GEO 관련 텍스트가 독자에게 보이지 않는가?
- [ ] AEO 관련 텍스트가 독자에게 보이지 않는가?
